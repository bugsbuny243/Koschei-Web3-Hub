KOSCH-bound API Referansı

API anahtarı yalnız kimlik bilgisidir; ürün erişimini atlamaz. Anahtarın sahibi doğrulanmış Basic-or-higher KOSCH holder olmalıdır.

Akış: Neon Auth hesabı → Phantom mesaj imzası → KOSCH bakiye doğrulaması → API anahtarı oluşturma → X-API-Key ile çağrı.

Token scan

curl -X POST https://tradepigloball.co/api/v1/scan/token \
  -H "X-API-Key: kos_live_..." \
  -H "Content-Type: application/json" \
  -d '{"mint":"SOLANA_MINT","network":"solana-mainnet"}'

Shield preflight

const response = await fetch('/api/v1/shield/preflight', {
  method: 'POST',
  headers: {'X-API-Key': apiKey, 'Content-Type': 'application/json'},
  body: JSON.stringify({target, intent: 'sign'})
});

Kullanım geçmişi

import requests
usage = requests.get(
  'https://tradepigloball.co/api/v1/usage',
  headers={'X-API-Key': api_key}
).json()

Erişim notu

Shopier, Paddle, paket satın alma veya ödeme webhook'u API erişimi vermez. API çağrılarında anahtar sahibinin KOSCH holder durumu yeniden doğrulanır.
