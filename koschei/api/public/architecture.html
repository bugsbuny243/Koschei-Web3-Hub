Koschei ARVIS
Technical architecture

Solana risk intelligence infrastructure.

ARVIS is organized as four layers: live source collection, evidence processing, final verdict generation and developer delivery. Every public metric below is read from the same production health source used by the live radar.

—completed stream targets
—signed verdicts in the proof window
—retryable backlog
—live launch and liquidity sources
Live production metrics · Values refresh every 15 seconds from the production health endpoint.

1. Source collection

ARVIS observes Solana launch and liquidity surfaces and stores normalized events for later risk processing.

2. Evidence processing

Independent evidence arms check launch behavior, liquidity context, wallet relations, authority state and claim risk.

3. Signed verdict

Verified evidence becomes a final verdict with grade, risk index, evidence list, rule version and signature state.

4. Developer delivery

The same verdict contract is used by radar screens, reports, partner APIs, watchlists and signed webhooks.

Production safety layers

Transaction simulation, Token-2022 extension analysis, bounded batch scans and RPC failover run as isolated Go services.

Ecosystem delivery

Wallet apps, launchpads, researchers and security teams can consume evidence through APIs, watchlist alerts and signed webhooks.
