# Contributing to Koschei Web3 Hub

Thank you for helping improve Koschei Web3 Hub. Contributions, feedback, bug reports, security copy improvements and documentation updates are welcome.

## Reporting Bugs

When reporting a bug, please include:

- A clear description of the issue.
- Steps to reproduce it.
- What you expected to happen.
- What actually happened.
- Browser, device and operating system details if relevant.
- Public wallet, token or transaction identifiers if they help reproduce the issue.

Do not include secrets, private keys, seed phrases, session tokens, passwords or sensitive personal information in issues.

## Suggesting Features

Feature suggestions are welcome, especially when they improve public-good Web3 safety and education. Please explain:

- The user problem.
- The proposed solution.
- Who benefits from it.
- Why it matters for Solana or Web3 users/builders.
- Any safety or education impact.

## Opening Pull Requests

Before opening a pull request:

- Keep the change focused and easy to review.
- Explain what changed and why.
- Include testing notes.
- Add screenshots for visible UI/UX changes.
- Avoid unrelated changes.
- Do not add secrets or credentials.
- Do not include private keys or seed phrases.

## Security and Privacy Rules

Never share or submit:

- Private keys.
- Seed phrases.
- API secrets.
- Session tokens.
- Passwords.
- Sensitive user data.

If you discover a security issue, follow the process in `SECURITY.md` instead of posting public exploit details.

## Focus Areas

Koschei especially welcomes contributions in these areas:

- Solana risk tooling.
- Wallet intelligence.
- Token scanning.
- Docs.
- UI/UX.
- Security copy.
