# Giveth Public Good Re-launch Strategy

## Project Name

**Koschei Public Shield: a free risk-prevention layer for Solana and Ethereum**

## Positioning

Koschei Public Shield should be relaunched on Giveth as a public-good project, not as a commercial SaaS product.

Core message:

> Koschei Public Shield is a non-profit, open-source, community-owned risk-prevention layer that helps Solana and Ethereum users detect MEV exposure, liquidity drains, and rug-pull patterns before losses happen.

## What Is Included in the Public Version

Only safety modules should be included in the public-good scope:

- **MEV Shield** — read-only transaction risk simulation and prevented-loss reporting.
- **Liquidity Radar** — liquidity-drain and rug-pull alert heuristics.
- **Public Impact Dashboard** — transparent aggregate metrics for prevented loss, protected wallets, and alert volume.
- **Community Risk Feed** — issue-based or pull-request-based submissions for suspicious pools, tokens, and transaction patterns.

## What Is Excluded

To keep the Giveth project clearly public-good and non-commercial, the public repository should exclude:

- Paid credit systems.
- Owner-only admin panels.
- Payment integrations.
- Private customer workflows.
- Commercial intelligence modules unrelated to public safety.
- Proprietary prompt chains or private operational dashboards.

## Giveth Project Description

**Short description:**

Koschei Public Shield is a free, open-source risk-prevention layer for Solana and Ethereum. It helps users and builders detect MEV risk, liquidity drains, and rug-pull patterns through non-custodial, read-only safety modules.

**Long description:**

Koschei Public Shield exists to make Web3 risk prevention measurable, accessible, and community-owned. Many users only discover MEV attacks, liquidity drains, or rug pulls after funds are lost. Koschei focuses on prevention: read-only analysis, transparent risk scoring, and public dashboards that show aggregate prevented-loss metrics.

The project is non-custodial and never asks for private keys, seed phrases, or custody. Its initial open-source modules are MEV Shield and Liquidity Radar, with public documentation and a community-maintained alert feed. Donations fund infrastructure, open-source maintenance, Solana/Ethereum integrations, documentation, and community education.

## Donation Call to Action

Donate to help keep Koschei Public Shield free for users, builders, and community security researchers. Each contribution funds open-source maintenance, public dashboards, RPC/indexing costs, documentation, and new prevention heuristics for Solana and Ethereum.

## Quadratic Funding Strategy

1. **Optimize for contributor count, not only donation size.** Ask many community members for small donations because QF rewards broad support.
2. **Publish proof before asking.** Share the `/impact` dashboard, screenshots, demo videos, and anonymized prevention logs.
3. **Create a simple donor ladder.** Suggested amounts: `$5`, `$10`, `$25`, and `$50+`.
4. **Coordinate launch week.** Announce on X, Farcaster, Discord, Telegram, Solana developer communities, Ethereum public-good channels, and Turkish Web3 groups.
5. **Ask builders to verify utility.** Encourage developers and security researchers to comment on Giveth explaining why read-only prevention tools matter.
6. **Post transparent updates.** Publish weekly updates with prevented-loss metrics, merged pull requests, new heuristics, and infrastructure spend.
7. **Convert donors into maintainers.** Invite donors to submit suspicious pool examples, documentation improvements, test cases, and chain-specific rules.

## First 14 Days Relaunch Plan

- **Day 1:** Create the `koschei-public` repository with MIT license, README, public shield scope, and contribution guide.
- **Day 2:** Publish the Giveth page with the public-good description and `/impact` dashboard link.
- **Day 3:** Post a founder thread explaining the deleted project reset and the new open-source-only scope.
- **Days 4-7:** Collect small donations from early users, Web3 friends, and developer communities.
- **Days 8-10:** Ship the first public issue list: MEV examples, liquidity-drain examples, documentation tasks, and good-first-issues.
- **Days 11-14:** Publish the first transparency update with dashboard metrics and donation usage plan.
